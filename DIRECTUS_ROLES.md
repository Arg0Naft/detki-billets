# Directus Roles

Ниже предложена базовая ролевая модель для self-hosted Directus.

## Administrator

Назначение:

- полный контроль над системой
- управление пользователями, ролями, расширениями и настройками

Права:

- полный доступ ко всем коллекциям и системным настройкам
- создание и удаление пользователей
- настройка публичных разрешений

## Content Manager

Назначение:

- наполнение сайта основным контентом мероприятия

Права:

- чтение и редактирование:
  - `site_settings`
  - `event_config`
  - `tickets`
  - `speakers`
  - `program`
  - `faq`
- без доступа к системным ролям и настройкам безопасности
- без удаления критичных системных сущностей

## Legal Editor

Назначение:

- поддержка юридического контента и ссылок

Права:

- чтение:
  - `site_settings`
  - `event_config`
- чтение и редактирование:
  - `legal_pages`
  - при необходимости текстов в `faq`, если туда выносятся юридические ответы
- без доступа к пользователям, ролям, расширениям и инфраструктурным настройкам

## Public read role

Назначение:

- анонимное чтение данных фронтендом

Права:

- `read` только для нужных коллекций:
  - `site_settings`
  - `event_config`
  - `tickets`
  - `speakers`
  - `program`
  - `faq`
  - `legal_pages`
- желательно ограничить список доступных полей до реально используемых фронтендом
- при возможности добавить фильтр по `is_active = true` для коллекций со списками

## Рекомендации

- Включить двухфакторную аутентификацию для администраторов.
- Не использовать роль администратора для повседневного редактирования контента.
- Проверить, что публичной роли недоступны системные коллекции Directus.
- Отдельно протестировать API-ответы от имени публичной роли перед запуском в продакшен.
